# GhostBot

Android-приложение (Java), которое подключается к твоему Bedrock-серверу как фейковый игрок
и переводит себя в режим спектатора (через команду `/gamemode spectator`, нужны права оператора).

## Как собрать APK онлайн (без ПК)

1. Создай новый репозиторий на GitHub, залей туда всю эту папку целиком.
2. Открой вкладку **Actions** в репозитории — сборка запустится автоматически при пуше в `main`
   (или нажми **Run workflow** вручную).
3. После завершения сборки зайди в конкретный run → внизу раздел **Artifacts** →
   скачай `GhostBot-debug` → внутри будет `app-debug.apk`.
4. Установи APK на телефон (разреши установку из неизвестных источников).

## Как это работает (авторизация)

Aternos требует Xbox Live авторизацию для всех удалённых подключений — `online-mode=false`
тут не помогает (это подтверждено официальной документацией Microsoft: настройка
работает только для LAN). Поэтому приложение делает то же, что и настоящий клиент Minecraft:

1. При запуске сервис выведет в уведомлении ссылку и код, например:
   `Открой https://www.microsoft.com/link и введи код: ABCD1234`
2. Открой эту ссылку в браузере (на любом устройстве), войди под своим Microsoft-аккаунтом
   (тем, на котором куплен/привязан Minecraft), введи код.
3. Приложение получит токен, обменяет его на подписанную Mojang цепочку сертификатов
   и подключится к серверу под этим аккаунтом.

Учти: бот будет заходить под твоим реальным аккаунтом (или отдельным, если укажешь его при входе) —
это как обычный игрок, просто управляемый программой.

## Известные риски первой сборки

Библиотека `org.cloudburstmc.protocol` иногда меняет имена классов/методов между версиями
(это активно разрабатываемый снапшот). Если сборка упадёт в Actions — пришли мне лог ошибки,
поправим конкретные места (как обычно с твоими другими проектами).

Также `protocolVersion` в `LoginPacket` (сейчас `766`) должен совпадать с версией Bedrock
на сервере — если после логина сразу дисконнект с "outdated client/server", поменяем это число.
